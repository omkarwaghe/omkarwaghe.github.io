Different Action types:

  • Add - Add current effective dated row only
  • Update/Display - Insert effective dated rows which are greater than the current and display current and future effective rows.
  • Include history - Insert effective dated rows which are greater than the current and display all historical, current and future effective rows.
  • Correct History - Add, change or delete historical, current and future effective dated rows.

Benefits Administration

  • Base Benefits: It enables us to setup the basic benefits system acrchitecture ad manually enroll employees and their dependents into the benefit plans.
  • Using Benefit Administration: It enables us to setup event and eligibility rules to automate the enrollment into benefit plan however, we will use base benefits for core system achitecture.

Base Benefits(Core system Architecture) Benefit Administration(Setup event and eligibility rules to facilitate automatic enrolments into benefit plans. If we are not using Benefits Admin then we can perform manual enrollments). Run Automated enrolment process

Base Benefits business process:

  1. Setting up supporting tables and benefits plans
  2. Bulding benefit programs
  3. Assigning employees to benefit programs and enroll them in plans for which their program enrollment makes them eligible
  4. Calculating rates and benefit deductions

Base benefits core Tables:

  1. Provider/Vender Table - Organization that provides or services a benefit offering
  2. Dependent Relationships - Defines Dependent Relationships such as child, grandpa
  3. Coverage Code - Defines who can be covered by the benefit for the health coverage
  4. Coverage Group Codes - Identifies the maximum lifetime coverage allowed in life and accidental death policies
  5. Service Step - Define a vesting schedule based on years of service for employer matching contributuons to savings plan
  6. Rate Tables - There are four types of rate tables: Age, Salary, Service and Flat. These tables are used to calculate the cost of benefit
  7. Calculation Rules - Define rules for calculating benefit deductions, such as as of dates for age, service, premium and coverage calculations.
  8. Deduction Code - Define the rules for calculating the amount of the deduction and how payroll should process the deduction
  9. Dependent Rules - It is used to establish age related limits on dependent eligibility for health plan coverage
  10. Benefit Plan Type - Any benefit Category
  11. Benefit plan A benefit offering within a plan type
  12. Benefit Program - A collection of benefit plans your company offers to a group of employees might be different for hourly, salaries and executive employees.

Setting up a benefit plans:

  • Benefit Plan Table: Setup HRMS->Product Related->Base Benefits->Plans and Providers->Benefit plan Table->Benefit plan Table.
  • Life and AD/D plan table: 20 is defined as Life in plan type. you can mention the insured person types as coverage employee or Dependent.
  • Defining Coverage Calculation Formulas: Set Up HCM-> Product Related -> Base Benefits -> Rates and Rules -> Coverage Formula Table -> Coverage Formula Table
  • Specify how the benefits compensation base is determined, a formula to apply against that base to derive a coverage amount and limit on the coverage amount.
  • Define whether the coverage is subject to reduction based on attained age.

Benefit Program: Setup HRMS ->Product Related -> Base Benefits -> Program Structure -> Benefit program table -> Benefit Program Program type: Automated then its using base benefits and Benefit administration installed if it's manual then only base benefit is available in the system and not benefit administration.

Common Elements Used for Benefits Administration

  • Benefit Program- A collection of benefit plans a company offers to a group of employees
  • Benefit Plan- Any benefit category such as health, life r savings. A benefit plan option is a specific benefit option within a plan type.
  • Event Maintainence - The process that is used to enrol new employees and to track and react to employe benefit changes during the plan year, such as resulting from marriage or termination of employment
  • Open Enrolmnt - The process hat is used to enroll or re-enrol participants in appropriate benefit programs and plans ducting the annual enrolment window or, for federal users, during the open season periods for health benefits, life plans and savings plans.
