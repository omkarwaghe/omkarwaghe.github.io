FieldDefault - Enables to set fields to default values when they are initially displayed. In add mode when the new row is inserted into a scroll area.
FieldEdit - used for Validation, until it does not get rectify component will not save
FieldChange - its for recalculating field values based on changes made to other fields.
PrePopup - Initiated just before the pop-up menu. used to control appearance of pop-up menu
SearchInit - used for SetSearchEdit, SetSearchDefault, ClearSearchEdit, SetSearchDialogBehavior
SearchSave -It triggers when clicked on Search button. used to write validation check whether any search criteria is entered on search page
RowInit - The RowInit event is initiated the first time that the Component Processor encounters a row of data
RowSelect - RowSelect PeopleCode is used to filter out rows of data as they are being read into the component buffer.
RowInsert - When a user adds a row of data, the Component Processor generates a RowInsert event
RowDelete - The RowDelete event is initiated whenever a user attempts to delete a row of data from a page scroll area
SaveEdit - Validation before save, recordA->recordB->ComponentRecordA->ComponentRecordB

SavePostChange
PreBulid - Triggered before the component build, used to hide unhide pages or used to set component variable
PostBuild - The PostBuild event is initiated after all the other component build events have been initiated. This event is often used to hide or unhide pages
SavePreChange - SavePreChange PeopleCode provides one final opportunity to manipulate data before the system updates the database

 Activate - The Activate event is initiated each time that a page is activated, including when a page is first displayed by a user, or if a user presses Tab between different pages in a component ItemSelected - The ItemSelected event is initiated whenever a user selects a menu item from a pop-up menu.
