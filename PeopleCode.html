Event Flow

Based on the Peoplesoft functionality we have certain peoplecode event which will be triggered when component will be loaded.

Record Field Event Component Record Field PeopleCode Event Component Record PeopleCode Event Component Event Page Event Menu Event
SearchInit - Triggered before the search dialogue box displays
PreBuild - Triggered before the component build, used to hide unhide pages or used to set component variable
FieldDefault - Enables to set fields to default values when they are initially displayed. In add mode when the new row is inserted into a scroll area.
FieldFormula - (it should not be used because) it initiates in many different contexts and triggers PeopleCode on every row in component buffer.
FieldEdit - used for Validation, until it does not get rectify component will not save
FieldChange - its for recalculating field values based on changes made to other fields.
RecordA->ComponentRecordA->recordB->ComponentRecordB
Prepopup - Initiated just before the pop-up menu. used to control appearance of pop-up menu
RowInit - The RowInit event is initiated the first time that the Component Processor encounters a row of data
RowSelect - RowSelect PeopleCode is used to filter out rows of data as they are being read into the component buffer.
RowDelete - The RowDelete event is initiated whenever a user attempts to delete a row of data from a page scroll area
RowInsert - When a user adds a row of data, the Component Processor generates a RowInsert event
SaveEdit - Validation before save, recordA->recordB->ComponentRecordA->ComponentRecordB
